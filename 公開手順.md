# 公開手順（GitHub Pages）

このフォルダ（`index.html` がルート）をそのまま GitHub Pages で公開する手順。
公開URL（予定）: **https://chiiita.github.io/iikae/**

> ⚠️ `git push` はサンドボックスからできないため、**お手元のターミナルで実行**してください。

---

## 1. リポジトリ名を決める
仮に `iikae`。別の名前にする場合は、`index.html` の中の **`chiiita.github.io/iikae`** を5か所すべて新しいリポジトリ名に置換してください（canonical / og:url / og:image / twitter:image）。

## 2. GitHubに新規リポジトリを作成
- ブラウザで github.com → New repository → Repository name: `iikae` → **Public** → Create
- （または gh CLI: `gh repo create iikae --public`）

## 3. このフォルダを push
ターミナルでこのフォルダに移動して：

```bash
cd "/Volumes/CIRAGO/クラウドコード/お悩み解決ツール/A_言い換えジェネレーター"
git init
git add -A
git -c user.name="chiiita" -c user.email="61923324+chiiita@users.noreply.github.com" commit -m "publish: 角が立たない言い換えメーカー"
git branch -M main
git remote add origin https://github.com/chiiita/iikae.git
git push -u origin main
```

> もし `/Volumes/CIRAGO` 上で git がうまく動かない場合は、フォルダをホームにコピーしてから実行：
> ```bash
> cp -R "/Volumes/CIRAGO/クラウドコード/お悩み解決ツール/A_言い換えジェネレーター" ~/iikae && cd ~/iikae
> ```
> （以降は同じ。公開後の更新も ~/iikae 側で編集→push、もしくは CIRAGO 側を編集して cp で同期）

## 4. GitHub Pages を有効化
- リポジトリ → **Settings → Pages**
- **Source: Deploy from a branch** → Branch: **main** / **/(root)** → Save
- 1〜2分待つと **https://chiiita.github.io/iikae/** で公開される

## 5. 確認
- 公開URLを開いて表示・検索・コピーが動くか確認
- SNSにURLを貼ると `ogp.png`（角を立てずに、伝える。）のカードが出る
  - 反映されない時は各SNSのデバッガでキャッシュ更新（FB: developers.facebook.com/tools/debug ／ X: Card Validator）

---

## 更新のしかた（公開後）
`data/phrases.js`（例文）や `index.html` を編集 → `git add -A && git commit -m "..." && git push` で自動反映（数十秒）。
